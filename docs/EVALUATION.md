# Evaluation & Automated Metrics

This document details the mathematical formulations and implementations of the evaluation metrics in the `evaluation` package.

---

## 1. BLEU (Bilingual Evaluation Understudy)

BLEU evaluates the quality of text generated by comparing candidate sentences to references.

### Formulation:
1. **Modified n-gram Precision ($P_n$):**
   $$P_n = \frac{\sum_{i} \sum_{g \in \text{ngrams}_n(c_i)} \min(\text{count}(g, c_i), \max_{j} \text{count}(g, r_{ij}))}{\sum_i \sum_{g \in \text{ngrams}_n(c_i)} \text{count}(g, c_i)}$$
   where $c_i$ is the candidate sentence and $r_{ij}$ are reference sentences.
2. **Brevity Penalty ($BP$):**
   $$BP = \begin{cases} 1 & \text{if } c > r \\ e^{1 - r/c} & \text{if } c \leq r \end{cases}$$
   where $c$ is the total candidate corpus length and $r$ is the reference corpus length.
3. **Final BLEU-N:**
   $$\text{BLEU}_N = BP \cdot \exp\left(\frac{1}{N} \sum_{n=1}^N \log P_n\right)$$

### Implementation Details:
* We compute **BLEU-1**, **BLEU-2**, **BLEU-3**, and **BLEU-4**.
* A smoothing function is implemented: if any n-gram match count is 0, precision is smoothed to `0.1 / total_ngrams` to prevent `log(0)` errors.

## 2. METEOR

METEOR aligns words in candidate and reference sentences and evaluates them based on unigram mapping.

### Steps:
1. **Alignment Mapping:** Matches unigrams between candidate and reference using:
   * **Stage 1:** Exact matching (words are identical).
   * **Stage 2:** Stem matching (using a Porter Stemmer fallback, matching base suffixes).
2. **Precision ($P$) & Recall ($R$):**
   $$P = \frac{m}{w_c}, \quad R = \frac{m}{w_r}$$
   where $m$ is the number of matched unigrams, $w_c$ is candidate word length, and $w_r$ is reference word length.
3. **F-mean:**
   $$F_{\text{mean}} = \frac{10 \cdot P \cdot R_e}{R_e + 9P}$$
   (Recall is weighted 9 times more than precision).
4. **Chunk Penalty:** To reward contiguous word orders:
   $$Pen = 0.5 \cdot \left(\frac{chunks}{m}\right)^3$$
   where $chunks$ is the number of contiguous matching blocks in the alignment.
5. **Final Score:**
   $$\text{METEOR} = F_{\text{mean}} \cdot (1 - Pen)$$

## 3. ROUGE-L (Longest Common Subsequence)

ROUGE-L measures the longest common subsequence of tokens between predictions and references.

### Formulation:
* Let $\text{LCS}(C, R)$ be the length of the longest common subsequence between candidate $C$ of length $m$ and reference $R$ of length $n$.
* **Precision & Recall:**
   $$P_{\text{lcs}} = \frac{\text{LCS}(C, R)}{m}, \quad R_{\text{lcs}} = \frac{\text{LCS}(C, R)}{n}$$
* **F-measure:**
   $$\text{ROUGE-L} = \frac{(1 + \beta^2) P_{\text{lcs}} R_{\text{lcs}}}{R_{\text{lcs}} + \beta^2 P_{\text{lcs}}}$$
   We use the standard MS COCO evaluation parameter: $\beta = 1.2$.

## 4. CIDEr (Consensus-based Image Description Evaluation)

CIDEr evaluates candidate captions by measuring their similarity to reference consensus using TF-IDF weighting.

### Formulation:
1. **Document Frequency (DF):** Counts how many images in the reference corpus contain a given n-gram $g$:
   $$\text{DF}_g = \sum_{i=1}^M \mathbb{1}(g \text{ occurs in reference } S_i)$$
2. **IDF Weighting:**
   $$\text{IDF}_g = \log\left(\frac{M}{\text{DF}_g}\right)$$
3. **Cosine Similarity:** Computes similarity of TF-IDF vectors for candidate $c_i$ and reference $s_{ij}$:
   $$\text{CIDEr}_n(c_i, s_{ij}) = \frac{\boldsymbol{g}(c_i) \cdot \boldsymbol{g}(s_{ij})}{\|\boldsymbol{g}(c_i)\| \|\boldsymbol{g}(s_{ij})\|} \cdot e^{-\frac{(l(c_i) - l(s_{ij}))^2}{2 \sigma^2}}$$
   where $e^{-\frac{(l(c_i) - l(s_{ij}))^2}{2 \sigma^2}}$ is a length penalty ($\sigma=6.0$).
4. **Final Score:** Average over $n=1..4$ scaled by 2.5:
   $$\text{CIDEr} = \sum_{n=1}^4 2.5 \cdot \text{CIDEr}_n(c_i, S_i)$$

## 5. VQA Accuracy Metrics

Visual Question Answering uses exact and string alignment metrics:

* **Exact Match (EM):** Checks if the preprocessed (lowercased, punctuation-removed) prediction is identical to the reference.
* **Token Accuracy (F1):** Computes token-level precision and recall intersections, returning their harmonic mean (F1 score).
* **Word Accuracy:** Calculates word-level Levenshtein edit distance $D$:
  $$\text{Word Accuracy} = 1.0 - \frac{D}{\max(|words_{\text{pred}}|, |words_{\text{ref}}|)}$$
